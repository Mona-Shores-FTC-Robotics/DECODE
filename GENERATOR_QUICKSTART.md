# .pp File Autonomous Generator - Quick Start

## What You Got

A complete system for generating command-based autonomous routines from `.pp` files with **granular control** over every aspect.

## Files Created

### Core System
1. **`PedroPathLoader.java`** - Loads and parses .pp files
2. **`AutoRoutineBuilder.java`** - Fluent API for building autonomous routines
3. **`DecodeAutoGeneratedExample.java`** - Proof-of-concept OpMode

### Assets
4. **`.pp files moved to assets/**
   - `TeamCode/src/main/assets/trajectory.pp`
   - `TeamCode/src/main/assets/short_auto_logic.pp`

### Documentation
5. **`docs/pp-file-auto-generator.md`** - Complete usage guide

## Quick Example

```java
private Command buildAutonomousRoutine() {
    return AutoRoutineBuilder
        .fromPpFile("trajectory.pp", robot, hardwareMap, activeAlliance)

        // Fast first segment with launcher spinning
        .followNext()
            .withPower(0.9)
            .during(spinUpLauncher())
        .andThen()

        // SLOW SECOND SEGMENT (as you requested!)
        .followNext()
            .withPower(0.5)  // Slow down!
        .andThen()

        // USE TANGENTIAL HEADING (as you requested!)
        .followNext()
            .withTangentialHeading()  // Instead of linear!
        .andThen()

        .build();
}
```

## Granular Controls Available

✅ **Per-segment speed:** `.withPower(0.5)` - slow down any path
✅ **Heading interpolation:** `.withLinearHeading()`, `.withTangentialHeading()`, `.withHeadingInterpolation(0.7)`
✅ **Actions before/during/after:** `.before()`, `.during()`, `.after()`
✅ **Follow by name:** `.followSegment("Path 3")` instead of order
✅ **Skip segments:** `.skip(2)` for testing
✅ **Insert commands:** `.then()`, `.delay()`

## Next Steps

### 1. Test the Example OpMode

Run `DecodeAutoGeneratedExample` on your robot:
- It loads `trajectory.pp` (already in assets)
- Shows all granular controls in action
- Edit `buildAutonomousRoutine()` to customize

### 2. Create Your Own Path

1. Use Pedro Pathing GUI to design path
2. Save as `.pp` file
3. Copy to `TeamCode/src/main/assets/your_path.pp`
4. Update `config.ppFileName = "your_path.pp"` in OpMode

### 3. Customize Actions

Replace example actions with your subsystem commands:
```java
.followNext()
    .during(launcherCommands.spinUpForPosition(FieldPoint.LAUNCH_FAR))
    .after(launcherCommands.launchAllAtRangePreset(LauncherRange.LONG, false))
.andThen()
```

## Real-World Use Cases

### "Slow down the 3rd path for accurate collection"
```java
.followNext().withPower(0.8).andThen()  // Path 1: Normal
.followNext().withPower(0.8).andThen()  // Path 2: Normal
.followNext().withPower(0.4).andThen()  // Path 3: SLOW!
.followNext().withPower(0.9).andThen()  // Path 4: Fast again
```

### "Use tangential heading on curves"
```java
.followNext()
    .withTangentialHeading()  // Follows the path curve naturally
    .withPower(0.8)
.andThen()
```

### "Parallel intake while driving"
```java
.followNext()
    .during(new InstantCommand(() ->
        robot.intake.setMode(IntakeSubsystem.IntakeMode.ACTIVE_FORWARD)))
    .withPower(0.7)
.andThen()
```
